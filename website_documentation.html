  To explain the website we first have the front homepage (index.html) which welcomes you with open arms. There is nothing special about this since it was just built to be a welcoming page.

  The next page Doctor (doctor.html) allows for the construction of ids based on some stats, while custon ids can be added, this will allow for a more professional id to be created. You can also utilize a LOAD, Save feature to have it remember previously created ids. I used this system instead of others because I originally didn't know how to interface directly with the database and, even after teaching myself how to in the last couple days before submission. I did not know how to properly complete this and allow you to see these interactions on your end without paying money to host the site or locally hosting it. This lead to compromising with this setup.

  Nurse (nurse.html) is the same as Doctor except it is for Nurse instead of Doctor.

  Patient (patient.html) is the same as Doctor and Nurse except it has more options to take into account an entity with more data being stored and with more potential variables to deal with.

  Documentation (database_documentation.html) has many sub menus to go through but the gist is that they go over the documentation that the name describes.
